# devops-netology

## Настройка .gitignore

В проекте настроен файл `.gitignore` для исключения из системы контроля версий файлов, которые не должны попадать в репозиторий:

# 1. Файлы конфигурации IDE и редакторов кода
- `.idea/`, `.vscode/` - служебные файлы JetBrains IDE и VS Code
- Исключаются, так как содержат персональные настройки рабочего окружения, которые не должны быть общими для всех разработчиков

# 2. Временные и системные файлы
- `*.tmp`, `*.temp` - временные файлы, создаваемые различными программами
- `.DS_Store` (macOS), `Thumbs.db` (Windows) - служебные файлы операционных систем
- Эти файлы не несут полезной информации для проекта и создаются автоматически

# 3. Файлы Terraform
Для Terraform игнорируются:

- **Файлы состояния** (`*.tfstate`, `*.tfstate.*`) - содержат конфиденциальную информацию и текущее состояние инфраструктуры, должны храниться в защищённом бэкенде
- **Файлы переменных с данными** (`*.tfvars`, `*.tfvars.json`) - могут содержать чувствительные данные (пароли, ключи)
- **Логи ошибок** (`crash.log`) - не нужны для работы приложения
- **Конфигурационные файлы CLI** (`.terraformrc`, `terraform.rc`) - персональные настройки пользователей
- **Кэш-каталоги** (`.terraform/`) - содержат скачанные провайдеры и модули, которые можно восстановить командой `terraform init`
- **Файлы переопределений** (`override.tf`, `*_override.tf`) - обычно используются для локальных изменений


Игнорирование этих файлов позволяет:
- Не засорять историю коммитов служебными файлами
- Избегать случайной публикации чувствительных данных (паролей, ключей доступа)
- Предотвращать конфликты при совместной работе над проектом
- Сохранять репозиторий чистым от файлов, зависящих от локального окружения
